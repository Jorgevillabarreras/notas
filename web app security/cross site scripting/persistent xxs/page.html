Los ataques XSS persistentes ocurren cuando el payload se envía a el servidor web vulnerable y luego se almacena.
Cuando una pagina web del site vulnerable extrae los archivos maliciosos almacenados en el código y lo coloca dentro de la salida HTML,
entregará el payload XSS.
Se llama persistente porque el código malicioso se entregado cada vez que un navegador web accede al Página web inyectada.

Esta es una forma muy peligrosa de XSS porque, con un solo ataque, el hacker puede explotar múltiples aplicaciones usuarios web.
Por ejemplo:
Si un atacante logra escribir una payload maliciosa (HTML o JavaScript) en una página de red social,
cada usuario que visita esa página ejecutará el payload!


El vector más común de ataques persistentes es los formularios HTML que envían contenido al servidor web y luego mostrar ese contenido a los usuarios.
Elementos como comentarios, perfiles de usuario y posts de foro son un vector potencial para los ataques XSS.
