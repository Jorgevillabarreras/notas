Cross Site Scripting (XXS)es una vulnerabilidad que deja al atacante controlar algunas partes de las aplicaciones web

Cuando usamos unXXS,el atacante puede:
  • Modificarel contenido del website run-time
  • injectarcontenido malisioso
  • Robar el Cookie,durante la sesion del usuario
  • hacer acciones en el web app como si fuera un user legitimo
  • y mucho mas

Los actores invuelto en un XXS attack son:
  • El site vulnerable
  • el usuario que es la victima(el que visita el site)
  • El penetration tester



XSS pasa cuando un web applicacion usaunfilteres user inputpara crear un output para ensenarselo al end user;
esto deja que atacante controle el output de HTML y Javascript, mientras ataca el usuario

En este tipo de ataque, el user inputes cualquier parametro procediente del lado del cliente al web app.
como:
  • Request headers
  • Cookie
  • Form inputs
  • POST parameters
  • GET parameters


Todo es estos input deben ser validados por el lado del servidor poniendo funcion de seguridad que sanitize o filtre la entrada del usuario
La mayoria del tiempo las victimas de XXS son los usuarios o visitantes del website.Ten en mente que unos de los usuarios podria ser el administrador

XXS injecta codigo maliciosoen el output de una pagina.El codigo malioso es despues ejecutado por el browser del usuario visitante

Muchos desarolladores consideran limpiar XXS unaprioridad baja;Por que cuando alguien encuentra XXS lo unico que hace es message windows
pero como vamos a ver un atacante puede hacer muchho mas!!

Si un web app es vulnerable a XXS puede ser bien dificil para la victima que se de cuenta que un ataque este en proceso
la mayoria del tiempo los ataques lo hacen inreconosible que la pagina tiene algun cambio

Los hacker exploit XXS attack haciendo que los usuarios:
  • Haciendo que el browser load contenido malicioso
  • Haciendo operaciones como si fueran el usuario,Como comprar algo o cambiar el password
  • Robando el session cookie, asi pueden hacerse pasar como si fueran el usuario

Hacerse pasar por el usuario puede causar que coja el control del website completo.Que pasa si roba la aunthenticacion del admin?

  • Cookie stealing
